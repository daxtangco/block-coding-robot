## Automatic Visualizations (YOLOv8)

These are **automatically generated** during training by YOLOv8.

### Location
```
training_output/models/experiment_X_TIMESTAMP/stageN_name/
```

### Graphs Included

#### 1. **results.png** - Training Progress Overview
**4-in-1 combined chart showing:**
- Training & validation box loss
- Training & validation classification loss
- mAP@0.5 progression
- mAP@0.5:0.95 progression

**Use for:** Overall training progress visualization

---

#### 2. **confusion_matrix.png** - Prediction Confusion Matrix
**Shows:**
- True positives for each class
- Misclassifications between classes
- Background (false positive) detections

**Use for:** Understanding which classes are confused with each other

---

#### 3. **confusion_matrix_normalized.png**
**Same as above but normalized** (percentages instead of counts)

**Use for:** Better comparison across imbalanced classes

---

#### 4. **F1_curve.png** - F1 Score vs Confidence Threshold
**Shows:**
- F1 score at different confidence thresholds
- Optimal confidence threshold (marked)

**Use for:** Selecting best confidence threshold for deployment

---

#### 5. **PR_curve.png** - Precision-Recall Curve
**Shows:**
- Precision-Recall trade-off
- Area under curve
- Per-class curves

**Use for:** Understanding model's detection quality

---

#### 6. **P_curve.png** - Precision vs Confidence
**Shows:**
- How precision changes with confidence threshold

---

#### 7. **R_curve.png** - Recall vs Confidence
**Shows:**
- How recall changes with confidence threshold

---

#### 8. **labels.jpg** - Training Data Distribution
**Shows:**
- Class distribution histogram
- Bounding box size distribution
- Bounding box position heatmap

**Use for:** Understanding dataset characteristics

---

#### 9. **labels_correlogram.jpg** - Label Correlations
**Shows:**
- Correlation between bounding box dimensions
- Position correlations

---

#### 10. **results.csv** - Raw Metrics Data
**Contains:**
- Per-epoch metrics (loss, mAP, P, R)
- All training statistics

## How to Generate All Graphs

### Step 1: During Training (Automatic)

YOLOv8 graphs are generated automatically:

```bash
python train.py --experiment 2
```

**Output location:**
```
training_output/models/experiment_2_TIMESTAMP/
├── stage1_synthetic/
│   ├── results.png
│   ├── confusion_matrix.png
│   ├── F1_curve.png
│   ├── PR_curve.png
│   └── results.csv
└── stage2_finetuned/
    ├── results.png
    ├── confusion_matrix.png
    └── ... (same files)
```

---

### Step 2: After Training (Validation Graphs)

Run validation to generate per-class and summary graphs:

```bash
python validate.py --model training_output/models/experiment_2_*/stage2_finetuned/weights/best.pt --experiment 2
```

**Output location:**
```
training_output/results/validation_best/
├── per_class_performance.png
├── metrics_summary.png
├── metrics.json
└── validation_report.txt
```

---

### Step 3: Thesis Results Graphs (After Both Experiments)

Generate comparison graphs for thesis:

```bash
python generate_thesis_graphs.py \
  --exp1-results training_output/models/experiment_1_*/stage1_synthetic/results.csv \
  --exp1-metrics training_output/results/validation_exp1/metrics.json \
  --exp2-results training_output/models/experiment_2_*/stage2_finetuned/results.csv \
  --exp2-metrics training_output/results/validation_exp2/metrics.json
```

**Output location:**
```
training_output/results/thesis_graphs/
├── fig1_training_loss_comparison.png
├── fig2_map_progression.png
├── fig3_precision_recall_progression.png
├── fig4_final_metrics_comparison.png
├── fig5_per_class_comparison.png
├── fig6_inference_speed_comparison.png
├── fig7_success_criteria_dashboard.png
└── results_table.md
```

## Complete Workflow Example

```bash
# 1. Train Experiment 1
python train.py --experiment 1

# 2. Validate Experiment 1
python validate.py \
  --model training_output/models/experiment_1_*/stage1_synthetic/weights/best.pt \
  --experiment 1

# 3. Benchmark Experiment 1
python test.py \
  --model training_output/models/experiment_1_*/stage1_synthetic/weights/best.pt \
  --image datasets/images/00000.jpg \
  --benchmark

# 4. Train Experiment 2
python train.py --experiment 2

# 5. Validate Experiment 2
python validate.py \
  --model training_output/models/experiment_2_*/stage2_finetuned/weights/best.pt \
  --experiment 2

# 6. Benchmark Experiment 2
python test.py \
  --model training_output/models/experiment_2_*/stage2_finetuned/weights/best.pt \
  --image datasets/images/00000.jpg \
  --benchmark

# 7. Generate thesis comparison graphs
python generate_thesis_graphs.py \
  --exp1-results <path-to-exp1-results.csv> \
  --exp1-metrics <path-to-exp1-metrics.json> \
  --exp2-results <path-to-exp2-results.csv> \
  --exp2-metrics <path-to-exp2-metrics.json>
```

## Tips for Thesis

1. **High Resolution**: All thesis graphs are saved at 300 DPI (publication quality)

2. **Color Scheme**: Consistent colors across all graphs for easy interpretation

3. **Annotations**: All graphs include:
   - Clear titles and labels
   - Target lines where applicable
   - Legends
   - Grid lines for readability

4. **Formats**:
   - PNG for inclusion in thesis
   - CSV data available for custom graphs
   - Markdown tables easily convert to LaTeX

5. **Customization**: Edit `generate_thesis_graphs.py` to:
   - Change colors
   - Adjust figure sizes
   - Add/remove metrics
   - Modify titles

## Quick Reference

**Question:** How do I get graphs for my thesis?

**Answer:**
1. Train models: `python train.py --experiment 1` and `--experiment 2`
2. Validate both: `python validate.py --model <path> --experiment X`
3. Generate comparisons: `python generate_thesis_graphs.py <args>`
4. Find graphs in: `training_output/results/thesis_graphs/`

**Total time:** ~5-30 hours (training) + 10 minutes (graph generation)
